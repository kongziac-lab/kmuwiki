### 5. 컷오버 게이트

다음 명령이 종료 코드 0과 `cutover.ready=true`를 반환하기 전에는 검색 환경을 변경하지 않는다.

```sh
python -m kmu_ingest.cli cutover-check \
  --expected-source-archives 22 \
  --minimum-total-documents 78 \
  --minimum-v2-documents 71
```

필수 성공 조건은 다음과 같다.

- 기존 검색 가능 문서 71개의 ID가 모두 v2에 존재하고 `processed_v1=0`
- 모든 v2 문서에 `search_units`가 하나 이상 존재
- 모든 v2 벡터가 `embed-v4.0/v4.0-1024` 하나의 모델로만 구성
- `ready` 시각 자산은 비공개 Storage 경로와 픽셀 마스킹 확인을 모두 보유
- 기존 v1 `doc_chunks`가 단일 모델 `embed-multilingual-v3.0/v3`로 보존

`pending_review`, `pending_ocr`, `blocked`, `failed` 시각 자산은 안전한 대체 표현만 검색한다는
경고다. 데이터 누락·모델 혼합·미마스킹 `ready` 자산은 실패이며 우회하지 않는다.

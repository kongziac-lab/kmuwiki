# 마스킹 품질 평가 하네스 (§7.A)

마스킹이 **가려야 할 PII를 다 가렸는지(recall=안전)** 와 **가리면 안 되는 값을
건드리지 않았는지(keep=활용성)** 를 수치로 검증하고, 임계값 미달 시 게이트를 실패시킨다.

## 실행
```bash
cd ingest
python -m evaluation.evaluate              # 운영 설정대로(NER 포함) 평가, 게이트
python -m evaluation.evaluate --no-ner     # 정규식만 베이스라인
python -m evaluation.evaluate --json out.json
```
종료코드: 게이트 통과 0 / 실패 1 → CI 게이트로 사용.

## 골든 데이터 (`golden/*.jsonl`)
한 줄당 한 문서. **전부 합성(가짜) — 실제 개인정보 저장 금지.**
```json
{"id":"g01","text":"...홍길동 900101-1234567...",
 "must_mask":[{"type":"성명","value":"홍길동"},{"type":"주민등록번호","value":"900101-1234567"}],
 "must_keep":["제2025-13호","2025-03-02"]}
```
- `must_mask` : 출력에서 사라져야 하는 PII(유형별). 남으면 **LEAK**.
- `must_keep` : 보존돼야 하는 비-PII(문서번호·날짜·기관명·금액). 사라지면 **OVER-MASK**.

## 임계값 (evaluate.py)
정형 식별자(주민/전화/이메일/카드) recall 1.0, 비정형(성명 0.95·주소 0.90), keep 0.98.

## 현재 결과 / 알려진 갭
- ✅ 정형 PII recall 1.0, 과다 마스킹 0 (계좌 규칙을 '키워드 인접 시에만'으로 수정해 날짜·문서번호 보존).
- ⚠️ **성명 recall이 NER 모델 컨텍스트에 따라 흔들린다**(예: 인접 토큰이 바뀌면 같은 이름을 놓침).
  게이트가 이를 정상적으로 차단한다. 운영 전 보완 필요:
  1. **골든셋 확대**(현재 10건 → 50~100건). 표본이 작아 임계값이 거칠다.
  2. **이름 재현율 보강 레이어**(성씨 가제티어 + 역할어 컨텍스트: "기안자/담당/예금주 + 이름").
  3. NER 모델 교체/앙상블 검토.
